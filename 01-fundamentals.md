# 01 - 设计基础概念

> 理解 UI 和 UX 的区别，建立正确的设计思维。

---

## UI vs UX 的区别

| 维度 | UI (用户界面) | UX (用户体验) |
|------|--------------|--------------|
| **定义** | 产品的外观和感觉 | 使用产品的整体体验 |
| **包含** | 按钮、颜色、字体、布局 | 流程、逻辑、情感、易用性 |
| **核心问题** | "看起来怎么样？" | "用起来怎么样？" |
| **学科** | 视觉设计 | 交互设计 |

### 形象比喻

```
UI = 皮肤（外表）
UX = 骨骼 + 神经系统（支撑 + 运作）
```

### 实际例子

**登录页面：**
- **UI**：蓝色按钮、圆角输入框、居中对齐的 Logo
- **UX**：错误时显示具体提示、记住用户名、支持回车登录

## 设计思维框架

### 基础流程

```
用户问题 → 理解需求 → 设计方案 → 原型测试 → 迭代优化
     ↑                                              ↓
     └────────────── 持续改进 ←─────────────────────┘
```

### 每个阶段的关键问题

| 阶段 | 关键问题 | 输出 |
|------|---------|------|
| **发现问题** | 用户遇到什么困难？ | 问题陈述 |
| **理解需求** | 用户真正需要什么？ | 用户画像、场景 |
| **设计方案** | 如何解决这个问题？ | 草图、原型 |
| **原型测试** | 方案是否有效？ | 测试结果 |
| **迭代优化** | 如何改进？ | 改进版本 |

---

## 以用户为中心的设计 (UCD)

### 核心原则

1. **了解用户** — 不是想象用户，而是研究真实用户
2. **早期测试** — 不要等到做完才测试
3. **迭代改进** — 设计是循环，不是直线

### 常见误区

| ❌ 错误做法 | ✅ 正确做法 |
|-----------|-----------|
| "我觉得用户会喜欢" | "用户测试数据显示..." |
| 先设计再考虑用户 | 先研究用户再设计 |
| 追求一次性完美 | 快速迭代，小步快跑 |

---

## 设计 vs 艺术

| 设计 | 艺术 |
|------|------|
| 解决问题 | 表达情感 |
| 有明确目标 | 自由创作 |
| 需要验证效果 | 不需要验证 |
| 为用户服务 | 为创作者服务 |

> 💡 **记住**：设计是功能性的，不是纯粹的美学追求。
