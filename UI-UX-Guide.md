## 设计基础概念

### UI vs UX 的区别

| UI (用户界面) | UX (用户体验) |
|--------------|--------------|
| 产品的外观和感觉 | 使用产品的整体体验 |
| 按钮、颜色、字体、布局 | 流程、逻辑、情感、易用性 |
| "看起来怎么样" | "用起来怎么样" |
| 视觉设计 | 交互设计 |

**简单记忆**：UI 是皮肤，UX 是骨骼和神经系统。

### 设计思维框架

```
用户问题 → 理解需求 → 设计方案 → 原型测试 → 迭代优化
```

## UI 设计核心原则

### 1. 视觉层次 (Visual Hierarchy)

**什么是它**：引导用户先看什么、后看什么的设计方法。

**如何实现**：
- **大小**：重要的元素更大
- **颜色**：重要的元素更醒目
- **对比**：高对比度吸引注意力
- **留白**：给重要元素周围留更多空间
- **位置**：屏幕上方和左侧通常更醒目

**Prompt 关键词**：
```
"清晰的视觉层次"
"重要信息突出显示"
"引导用户视线从上到下"
"主按钮使用高对比色"
```

### 2. 一致性 (Consistency)

**什么是它**：相同的功能看起来、用起来都一样。

**为什么重要**：减少用户学习成本，建立信任感。

**需要一致的元素**：
- 按钮样式（主按钮、次按钮、危险按钮）
- 字体大小和颜色
- 间距系统
- 圆角大小
- 图标风格

**Prompt 关键词**：
```
"保持一致的设计风格"
"使用统一的按钮样式"
"遵循设计系统规范"
"相同的交互模式"
```

### 3. 对齐 (Alignment)

**什么是它**：元素之间的位置关系。

**基本规则**：
- 左对齐：适合阅读（西方语言）
- 居中对齐：适合标题、短文本
- 右对齐：适合数字、价格
- 两端对齐：适合正式文档

**Prompt 关键词**：
```
"左对齐文本"
"居中对齐的标题"
"网格对齐的布局"
"整齐的排版"
```

### 4. 亲密性 (Proximity)

**什么是它**：相关的元素靠近，不相关的元素分开。

**应用示例**：
- 表单标签靠近输入框
- 按钮组之间保持小间距
- 不同模块之间保持大间距

**Prompt 关键词**：
```
"相关的元素分组"
"清晰的模块划分"
"适当的间距层次"
"信息分组明确"
```

### 5. 对比 (Contrast)

**什么是它**：让元素之间产生差异，突出重要内容。

**对比类型**：
- 颜色对比（深色背景 + 浅色文字）
- 大小对比（大标题 + 小正文）
- 粗细对比（粗体标题 + 常规正文）
- 形状对比（圆角卡片 + 直角按钮）

**Prompt 关键词**：
```
"高对比度的配色"
"大小对比突出标题"
"清晰的视觉区分"
"重要元素使用强调色"
```

## UX 用户体验设计

### 1. 用户旅程 (User Journey)

**什么是它**：用户完成一个任务的完整流程。

**设计步骤**：
1. 用户目标是什么？
2. 需要哪些步骤？
3. 每一步需要什么信息？
4. 可能的错误和解决方案？
5. 成功后的反馈？

**Prompt 关键词**：
```
"清晰的用户流程"
"减少操作步骤"
"明确的下一步指引"
"流畅的交互体验"
```

### 2. 信息架构 (Information Architecture)

**什么是它**：组织和结构化信息的方式。

**核心问题**：
- 用户来找什么？
- 如何分类最合理？
- 如何导航最方便？

**常用结构**：
- 层级结构（树状）
- 矩阵结构（多维度筛选）
- 线性结构（步骤流程）

**Prompt 关键词**：
```
"清晰的信息架构"
"直观的导航结构"
"合理的分类组织"
"易于查找的布局"
```

### 3. 可用性原则 (Usability)

**尼尔森十大可用性原则**（简化版）：

1. **系统状态可见**：告诉用户发生了什么
2. **贴近真实世界**：用用户熟悉的语言
3. **用户控制**：允许撤销操作
4. **一致性**：遵循平台惯例
5. **防错设计**：预防错误发生
6. **识别而非回忆**：让用户看到选项，而非记住
7. **灵活高效**：兼顾新手和专家
8. **美观简洁**：去除无关信息
9. **错误提示**：清晰的错误信息和解决方案
10. **帮助文档**：易于搜索和查找

**Prompt 关键词**：
```
"清晰的反馈提示"
"错误预防设计"
"直观的操作方式"
"简洁的界面"
```

### 4. 无障碍设计 (Accessibility)

**什么是它**：让所有人都能使用你的产品。

**关键要点**：
- 颜色对比度至少 4.5:1
- 不要只用颜色传达信息
- 支持键盘操作
- 为图片添加 alt 文本
- 字体大小至少 16px

**Prompt 关键词**：
```
"无障碍设计"
"高对比度配色"
"清晰的文字可读性"
"支持键盘导航"
```

## 设计系统与组件

### 常见组件类型

#### 1. 导航组件
- **顶部导航栏**：主导航，通常包含 Logo、菜单、用户头像
- **侧边栏**：次级导航，适合工具类应用
- **标签页 (Tabs)**：同一层级的内容切换
- **面包屑**：显示当前位置和返回路径
- **分页器**：大量内容的分页展示

**Prompt 关键词**：
```
"顶部导航栏包含 Logo 和菜单"
"左侧边栏导航"
"标签页切换不同内容"
"面包屑导航显示路径"
```

#### 2. 输入组件
- **文本输入框**：单行文字输入
- **文本域**：多行文字输入
- **下拉选择**：从预设选项中选择
- **单选框**：多选一
- **复选框**：多选多
- **开关**：二选一（开/关）
- **日期选择器**：选择日期
- **文件上传**：上传文件

**Prompt 关键词**：
```
"表单包含文本输入和下拉选择"
"使用单选框进行选项选择"
"开关控制功能启用"
"日期选择器选择时间范围"
```

#### 3. 展示组件
- **卡片**：信息分组展示
- **列表**：垂直排列的信息
- **表格**：结构化数据展示
- **折叠面板**：可展开/收起的内容
- **标签**：状态或分类标识
- **头像**：用户头像展示
- **徽章**：数字或状态提示

**Prompt 关键词**：
```
"卡片式布局展示内容"
"表格展示结构化数据"
"折叠面板组织信息"
"标签标识不同状态"
```

#### 4. 反馈组件
- **提示消息 (Toast)**：短暂显示的操作反馈
- **对话框 (Modal)**：需要用户确认的重要信息
- **加载状态**：操作进行中的提示
- **进度条**：显示完成进度
- **空状态**：无数据时的友好提示
- **错误页面**：404 或其他错误提示

**Prompt 关键词**：
```
"操作成功后显示 Toast 提示"
"重要操作使用确认对话框"
"加载时显示骨架屏"
"空状态提供引导操作"
```

### 组件状态

每个交互组件通常有多种状态：

- **默认 (Default)**：正常状态
- **悬停 (Hover)**：鼠标移上去
- **聚焦 (Focus)**：键盘选中
- **激活 (Active)**：点击中
- **禁用 (Disabled)**：不可用
- **加载 (Loading)**：处理中
- **错误 (Error)**：出错状态

**Prompt 关键词**：
```
"按钮包含悬停和点击状态"
"输入框有聚焦状态样式"
"禁用状态降低透明度"
"加载状态显示动画"
```

## 配色与排版

### 配色基础

#### 色彩模式
- **HEX**：#FF5733（网页最常用）
- **RGB**：rgb(255, 87, 51)
- **HSL**：hsl(9, 100%, 60%)（最直观）

#### 配色方案类型

1. **单色 (Monochromatic)**
   - 同一色相的不同明度
   - 安全、专业、简洁

2. **类比色 (Analogous)**
   - 色轮上相邻的颜色
   - 和谐、自然

3. **互补色 (Complementary)**
   - 色轮上相对的颜色
   - 对比强烈、醒目

4. **三色 (Triadic)**
   - 色轮上等距的三种颜色
   - 丰富、平衡

#### 色彩角色

```
主色 (Primary)：品牌主色，用于主要操作
辅色 (Secondary)：辅助色，用于次要操作
强调色 (Accent)：突出特殊元素
背景色 (Background)：页面背景
表面色 (Surface)：卡片、面板背景
文字色 (Text)：正文和标题
错误色 (Error)：错误提示
成功色 (Success)：成功提示
警告色 (Warning)：警告提示
```

**Prompt 关键词**：
```
"使用蓝色作为主色"
"浅色背景配深色文字"
"绿色表示成功，红色表示错误"
"整体配色简洁专业"
```

### 排版基础

#### 字体分类

1. **无衬线字体 (Sans-serif)**
   - 现代、简洁、屏幕友好
   - 示例：Inter, Roboto, Helvetica, Arial

2. **衬线字体 (Serif)**
   - 传统、正式、易读（印刷）
   - 示例：Merriweather, Georgia, Times New Roman

3. **等宽字体 (Monospace)**
   - 每个字符宽度相同
   - 用于代码显示
   - 示例：Fira Code, JetBrains Mono

#### 字体大小层级

```
标题 1 (H1)：32-48px — 页面主标题
标题 2 (H2)：24-32px — 章节标题
标题 3 (H3)：18-24px — 小节标题
正文 (Body)：14-16px — 主要内容
小字 (Small)：12-14px — 辅助信息
```

#### 行高与字重

- **行高 (Line Height)**：1.5-1.6 适合正文，1.2-1.3 适合标题
- **字重 (Font Weight)**：
  - 400：常规 (Regular)
  - 500：中等 (Medium)
  - 600：半粗 (Semi-bold)
  - 700：粗体 (Bold)

**Prompt 关键词**：
```
"使用 Inter 无衬线字体"
"标题 32px 加粗，正文 16px 常规"
"正文行高 1.6"
"清晰的字体层级"
```

## 响应式设计

### 断点 (Breakpoints)

常见设备断点：

```
手机：< 640px
平板：640px - 1024px
桌面：1024px - 1280px
大屏：> 1280px
```

### 响应式策略

1. **移动优先 (Mobile First)**
   - 先设计手机版，再适配大屏
   - 推荐方式

2. **桌面优先 (Desktop First)**
   - 先设计桌面版，再适配小屏

### 布局适配

- **网格系统**：12列网格最常用
- **弹性布局**：内容自动调整
- **隐藏/显示**：小屏隐藏次要内容
- **堆叠布局**：小屏垂直排列，大屏水平排列

**Prompt 关键词**：
```
"响应式设计，适配手机和平板"
"移动优先的设计"
"小屏时垂直堆叠布局"
"大屏时多列并排显示"
```

## AI Prompt 设计技巧

### 1. 结构化 Prompt 模板

```
【角色】你是一位专业的 UI/UX 设计师
【任务】设计一个 [具体功能/页面]
【目标用户】[描述目标用户]
【风格】[设计风格描述]
【要求】
- [具体要求 1]
- [具体要求 2]
- [具体要求 3]
【参考】[如有参考图或链接]
```

### 2. 描述设计风格的词汇

#### 整体风格
```
现代简约 (Modern Minimalist)
玻璃拟态 (Glassmorphism)
新拟态 (Neumorphism)
扁平化 (Flat Design)
拟物化 (Skeuomorphism)
暗黑模式 (Dark Mode)
企业级 (Enterprise/Corporate)
活泼年轻 (Playful/Youthful)
高端奢华 (Premium/Luxury)
工业风 (Industrial)
```

#### 视觉感受
```
清爽干净 (Clean)
温暖友好 (Warm & Friendly)
专业可信 (Professional & Trustworthy)
科技感 (Tech-forward)
有机自然 (Organic/Natural)
大胆前卫 (Bold/Avant-garde)
柔和优雅 (Soft/Elegant)
```

#### 布局描述
```
居中对齐 (Center-aligned)
左对齐 (Left-aligned)
全宽布局 (Full-width)
固定宽度 (Fixed-width)
侧边栏布局 (Sidebar layout)
卡片网格 (Card grid)
分屏布局 (Split-screen)
单页滚动 (Single-page scroll)
```

### 3. 常见场景 Prompt 示例

#### 登录页面
```
设计一个现代简约风格的登录页面：
- 居中卡片式布局
- 深色背景配浅色卡片
- 包含邮箱/密码输入框
- 主按钮使用品牌蓝色
- 添加"忘记密码"链接
- 支持社交账号登录选项
- 整体风格专业可信
```

#### 仪表盘
```
设计一个数据分析仪表盘：
- 左侧固定导航栏
- 顶部显示关键指标卡片
- 中间区域放置图表
- 使用简洁的配色方案
- 数据可视化清晰易读
- 支持暗黑模式切换
```

#### 电商产品页
```
设计一个电商产品详情页：
- 左侧产品图片，右侧信息
- 显示价格、评分、库存状态
- 尺码/颜色选择器
- 加入购物车按钮突出
- 产品描述和规格标签页
- 相关商品推荐区域
- 整体风格温暖友好
```

### 4. Prompt 优化技巧

**❌ 不好的描述**：
```
"做一个好看的网页"
"设计一个 app"
"要现代化一点"
```

**✅ 好的描述**：
```
"设计一个 SaaS 产品的落地页，目标用户是中小企业主，
风格要专业可信但不过于严肃，使用蓝色和白色为主色调，
包含 Hero 区域、功能特性、定价表格和客户评价模块"
```

**优化要点**：
1. **具体**：说明用途、用户、场景
2. **量化**：使用具体数值（大小、间距、颜色）
3. **参考**：提供类似产品或风格参考
4. **分层**：按重要性列出需求
5. **约束**：说明不要什么同样重要

## 常用设计术语速查

| 术语 | 英文 | 含义 |
|-----|------|------|
| 留白 | White Space | 元素之间的空白区域 |
| 边距 | Margin | 元素外部的间距 |
| 内边距 | Padding | 元素内部的间距 |
| 栅格 | Grid | 页面布局的网格系统 |
| 圆角 | Border Radius | 边角的弯曲程度 |
| 阴影 | Shadow | 元素下方的阴影效果 |
| 渐变 | Gradient | 颜色平滑过渡 |
| 透明度 | Opacity | 元素的透明程度 |
| 悬停 | Hover | 鼠标移上去的状态 |
| 点击态 | Active | 点击时的状态 |
| 禁用 | Disabled | 不可用的状态 |
| 模态框 | Modal | 弹出式对话框 |
| 下拉菜单 | Dropdown | 点击展开的菜单 |
| 工具提示 | Tooltip | 悬停显示的小提示 |
| 骨架屏 | Skeleton | 加载时的占位动画 |
| 面包屑 | Breadcrumb | 显示当前位置的导航 |
| 分页 | Pagination | 内容分页导航 |
| 标签页 | Tabs | 切换内容的标签 |
| 手风琴 | Accordion | 可展开收起的内容 |
| 轮播图 | Carousel | 滑动展示的图片 |
