# 07 - AI Prompt 设计技巧

> 学会写出让 AI 理解并执行的设计指令。

---

## 结构化 Prompt 模板

### 通用模板

```
【角色】你是一位专业的 UI/UX 设计师
【任务】设计一个 [具体功能/页面]
【目标用户】[描述目标用户]
【风格】[设计风格描述]
【要求】
- [具体要求 1]
- [具体要求 2]
- [具体要求 3]
【参考】[如有参考图或链接]
```

### 简化模板

```
设计一个 [风格] 的 [页面/组件]，
用于 [用途]，目标用户是 [用户描述]。
使用 [主色] 和 [辅色] 配色，
包含 [具体元素]，
要求 [具体要求]。
```

## 常见场景 Prompt 示例

### 登录页面

```
设计一个现代简约风格的登录页面：
- 居中卡片式布局
- 深色背景配浅色卡片
- 包含邮箱/密码输入框
- 主按钮使用品牌蓝色
- 添加"忘记密码"链接
- 支持社交账号登录选项
- 整体风格专业可信
- 响应式设计，适配手机
```

### 仪表盘

```
设计一个数据分析仪表盘：
- 左侧固定导航栏，可折叠
- 顶部显示 4 个关键指标卡片
- 中间区域放置折线图和饼图
- 使用蓝色主色，白色背景
- 数据可视化清晰易读
- 支持暗黑模式切换
- 响应式，平板和桌面适配
```

### 电商产品页

```
设计一个电商产品详情页：
- 左侧产品图片（可放大），右侧信息
- 显示价格、评分、库存状态
- 尺码/颜色选择器
- 加入购物车按钮突出（橙色）
- 产品描述和规格标签页
- 相关商品推荐区域（横向滚动）
- 整体风格温暖友好
- 手机端图片和信息垂直排列
```

### 设置页面

```
设计一个应用设置页面：
- 左侧设置分类导航
- 右侧设置项列表
- 使用开关、下拉、输入框等组件
- 分组显示相关设置
- 保存按钮固定在底部
- 简洁的企业级风格
- 修改后有未保存提示
```

### 空状态页面

```
设计一个空状态页面：
- 居中显示插图（404 或空数据）
- 友好的说明文字
- 提供返回首页或创建按钮
- 使用品牌蓝色作为按钮色
- 整体风格简洁不压抑
```

## Prompt 优化技巧

### ❌ 不好的描述

```
"做一个好看的网页"
"设计一个 app"
"要现代化一点"
"颜色好看一些"
"布局好看点"
```

**问题**：太模糊，AI 无法理解你的需求

---

### ✅ 好的描述

```
"设计一个 SaaS 产品的落地页，
目标用户是中小企业主，
风格要专业可信但不过于严肃，
使用蓝色和白色为主色调，
包含 Hero 区域、功能特性、定价表格和客户评价模块，
要求响应式设计，适配手机和桌面"
```

**优点**：具体、可执行

## 优化要点

### 1. 具体 (Specific)

| ❌ 模糊 | ✅ 具体 |
|--------|--------|
| "好看的颜色" | "蓝色主色，白色背景" |
| "大一点的标题" | "标题 32px，正文 16px" |
| "现代风格" | "现代简约，圆角卡片，轻微阴影" |

### 2. 量化 (Quantifiable)

```
"圆角 8px"
"间距 16px"
"最大宽度 1200px"
"断点：640px 和 1024px"
```

### 3. 参考 (Reference)

```
"类似 Stripe 的官网风格"
"参考 Apple 的设计语言"
"像 Notion 一样简洁"
```

### 4. 分层 (Prioritize)

```
【必须】
- 响应式设计
- 暗黑模式支持

【应该】
- 动画效果
- 微交互

【可选】
- 自定义主题
```

### 5. 约束 (Constraints)

```
"不要使用渐变"
"避免过多的颜色"
"不要圆角超过 16px"
"不要使用衬线字体"
```

## 进阶技巧

### 使用设计系统语言

```
"使用 8px 网格系统"
"遵循 Material Design 规范"
"使用原子设计方法论"
"建立设计令牌（Design Tokens）"
```

### 描述交互细节

```
"按钮悬停时颜色变深 10%"
"卡片悬停时上浮 4px 并增加阴影"
"页面切换使用淡入淡出动画，300ms"
"加载时使用骨架屏占位"
```

### 考虑边界情况

```
"空状态时显示友好提示"
"错误时用红色边框和文字提示"
"加载中显示旋转动画"
"长文本截断并显示省略号"
```
