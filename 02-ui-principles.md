# 02 - UI 设计核心原则

> 掌握这 5 个原则，让你的界面设计立即提升一个档次。每个原则都包含文字示意图、常见错误和实用 Prompt 关键词。

---

## 1. 视觉层次 (Visual Hierarchy)

### 什么是它？

视觉层次是引导用户视线的设计方法——决定用户**先看什么、后看什么、忽略什么**。好的视觉层次让用户在 3 秒内找到页面最重要的信息。

### 实现方法

| 方法 | 说明 | 示例 |
|------|------|------|
| **大小** | 重要的元素更大 | 标题 > 副标题 > 正文 |
| **颜色** | 重要的元素更醒目 | 主按钮用品牌色，次要按钮用灰色 |
| **对比** | 高对比度吸引注意力 | 深色文字配浅色背景 |
| **留白** | 给重要元素周围留更多空间 | 标题上下留白 32px，正文行距 1.5 |
| **位置** | 屏幕上方和左侧更醒目 | 核心功能放左上角（符合 F 型阅读模式） |
| **字重** | 粗体更容易被注意到 | 标题 Bold 700，正文 Regular 400 |

### 文字示意：F 型阅读模式

用户浏览网页时，视线通常呈 F 型移动：

```
┌──────────────────────────────────┐
│ ████████████████████████████     │  ← 第一行：从左到右完整扫描
│ ██████████████████               │  ← 第二行：扫描范围缩短
│ █████████                        │
│ ████████████                     │
│ ██████                           │  ← 之后：沿左侧快速向下扫描
│ █████████                        │
│ ████                             │
└──────────────────────────────────┘
  最重要的信息放这里 →  左上角区域
```

### Before & After

**Before（没有层次）：**
```
┌──────────────────────────────────┐
│  欢迎来到我们的产品              │  16px 常规
│  这是一个帮助你管理任务的工具    │  16px 常规
│  立即开始  了解更多              │  16px 常规
│  已有超过10000名用户使用         │  16px 常规
└──────────────────────────────────┘
  ↑ 所有文字大小、颜色一样，没有焦点
```

**After（清晰的层次）：**
```
┌──────────────────────────────────┐
│                                  │
│  欢迎来到我们的产品              │  32px 粗体 #1A1A1A
│  这是一个帮助你管理任务的工具    │  18px 常规 #666666
│                                  │
│  [■ 立即开始]   了解更多 →       │  按钮品牌色  链接灰色
│                                  │
│  已有超过10000名用户使用         │  14px 常规 #999999
└──────────────────────────────────┘
  ↑ 大标题→副标题→主按钮→辅助链接→补充信息
```

### 常见错误

| 错误 | 为什么是错的 | 怎么修 |
|------|-------------|--------|
| 所有文字一样大 | 用户不知道先看哪里，信息平铺无主次 | 建立 3-4 级字号梯度（如 32/24/18/14px） |
| 太多"重要"元素 | 什么都突出 = 什么都不突出 | 每个页面只有 1 个主要行动点（CTA） |
| 留白不够 | 信息堆叠，视线无处休息 | 重要元素周围增加 1.5-2 倍间距 |
| 颜色层级混乱 | 次要信息用了醒目颜色，抢了主要信息的注意力 | 主要信息用高饱和色，次要信息用灰色/低饱和色 |

### Prompt 关键词

```
"清晰的视觉层次"
"重要信息突出显示"
"引导用户视线从上到下"
"主按钮使用高对比色，次要按钮使用灰色"
"标题使用更大字号和粗体"
"F型阅读模式布局"
"每个页面只有一个主要 CTA"
"通过留白创造层次感"
"字号梯度：标题 32px，副标题 24px，正文 16px"
```

## 2. 一致性 (Consistency)

### 什么是它？

相同的功能在**任何地方**看起来、用起来都一样。一致性是用户信任感的基石——当界面行为可预测，用户才能把注意力放在任务本身而非界面操作上。

### 一致性的三个层面

| 层面 | 含义 | 示例 |
|------|------|------|
| **视觉一致性** | 相同元素外观一致 | 所有主按钮都是蓝色圆角 |
| **功能一致性** | 相同操作行为一致 | 点击 "×" 总是关闭 |
| **外部一致性** | 符合平台约定 | iOS 返回在左上角，Android 在左下角 |

### 需要一致的元素

| 元素 | 一致的内容 | 具体规范示例 |
|------|-----------|-------------|
| **按钮** | 主按钮、次按钮、危险按钮的样式 | 主=蓝色填充，次=灰色边框，危险=红色 |
| **字体** | 大小、颜色、字重 | H1=32px/Bold, H2=24px/Semibold, Body=16px/Regular |
| **间距** | 8px 倍数系统 | 8, 16, 24, 32, 48, 64 |
| **圆角** | 统一使用固定值 | 按钮 8px，卡片 12px，输入框 6px |
| **图标** | 线型或面型，统一风格 | 全部用 24px 线型图标，2px 描边 |
| **动效** | 过渡时长和缓动函数 | 全部用 200ms ease-in-out |
| **阴影** | 统一的层级阴影 | 卡片: 0 2px 8px rgba(0,0,0,0.1) |

### Before & After

**Before（不一致）：**
```
  页面 A                        页面 B
┌──────────────┐              ┌──────────────┐
│              │              │              │
│ [■ 保存]     │  蓝色/8px圆角 │ [ 保存 ]     │  绿色/4px圆角
│              │              │              │
│ [删除]       │  灰色文字链接  │ [■ 删除]     │  红色填充按钮
│              │              │              │
│ 间距 16px    │              │ 间距 12px    │
└──────────────┘              └──────────────┘
  ↑ 用户困惑：为什么同样的操作，外观不同？
```

**After（一致）：**
```
  页面 A                        页面 B
┌──────────────┐              ┌──────────────┐
│              │              │              │
│ [■ 保存]     │  蓝色/8px圆角 │ [■ 保存]     │  蓝色/8px圆角
│              │              │              │
│ [  删除  ]   │  红色/边框    │ [  删除  ]   │  红色/边框
│              │              │              │
│ 间距 16px    │              │ 间距 16px    │
└──────────────┘              └──────────────┘
  ↑ 相同操作 = 相同外观 = 用户不用重新学习
```

### 常见错误

| 错误 | 为什么是错的 | 怎么修 |
|------|-------------|--------|
| 不同页面的按钮样式不同 | 增加认知负担，用户要重新判断按钮用途 | 定义按钮规范：主/次/危险三种样式 |
| 间距随意使用 | 页面看起来不整齐，缺乏专业感 | 严格使用 8px 倍数间距系统 |
| 混用图标风格 | 线型和填充型混用显得杂乱 | 选定一种风格全局统一 |
| "创新"过度 | 为了每个页面"独特"打破规范 | 一致性优先于创新，在规范内寻找变化 |

### Prompt 关键词

```
"保持一致的设计风格"
"使用统一的按钮样式：主按钮、次按钮、危险按钮"
"遵循设计系统规范"
"相同的交互模式"
"统一的间距系统（8px 倍数）"
"统一的圆角、阴影和动效"
"视觉语言保持一致"
"使用同一套图标风格"
```

## 3. 对齐 (Alignment)

### 什么是它？

对齐是让元素在**看不见的线**上排列整齐。即使用户看不到网格线，也能感受到对齐带来的秩序感和专业感。页面上**没有任何元素应该看起来是随意摆放的**。

### 对齐方式选择

| 方式 | 适用场景 | 示例 | 注意事项 |
|------|---------|------|---------|
| **左对齐** | 大段文字阅读 | 文章正文、表单标签 | 最常用，符合从左到右阅读习惯 |
| **居中对齐** | 标题、短文本 | 页面标题、按钮文字、空状态 | 仅用于短文本，长段落不要居中 |
| **右对齐** | 数字、价格 | 表格数字列、金额 | 便于数字位数对比 |
| **两端对齐** | 正式文档 | 印刷品、合同 | 网页中慎用，容易产生不均匀间距 |

### 文字示意：隐形网格线

```
  对齐                              不对齐
  │
  │ 用户名         │                  用户名
  │ ┌────────────┐ │               ┌────────────┐
  │ │            │ │                │            │
  │ └────────────┘ │               └────────────┘
  │ 密码           │                    密码
  │ ┌────────────┐ │                  ┌────────────┐
  │ │            │ │                  │            │
  │ └────────────┘ │                  └────────────┘
  │ [■ 登录]       │               [■ 登录]
  │                │
  ↑ 所有元素左边缘              ↑ 每个元素起始位置不同
    在同一条线上                  看起来凌乱不专业
```

### Before & After

**Before（对齐混乱）：**
```
┌──────────────────────────────────┐
│      产品名称                    │  居中
│ 价格：¥99                       │  左对齐
│          立即购买                │  偏右
│ 产品描述：这是一个很好的产品     │  左对齐（不同起点）
└──────────────────────────────────┘
```

**After（统一对齐）：**
```
┌──────────────────────────────────┐
│ 产品名称                         │  ← 左对齐
│ ¥99                              │  ← 左对齐
│ 这是一个很好的产品               │  ← 左对齐
│                                  │
│ [■ 立即购买]                     │  ← 左对齐
└──────────────────────────────────┘
  ↑ 所有元素共享一条左边缘线
```

### 常见错误

| 错误 | 为什么是错的 | 怎么修 |
|------|-------------|--------|
| 表单标签有的左对齐，有的右对齐 | 破坏视觉一致性，增加阅读负担 | 统一所有标签为左对齐 |
| 长段落居中对齐 | 每行起始位置不同，阅读困难 | 超过 2 行的文字一律左对齐 |
| 没有使用网格系统 | 元素位置靠"目测"，微妙地不齐 | 使用 12 列网格或 4/8px 基础网格 |
| 不同卡片内容对齐方式不一致 | 同一列卡片看起来不统一 | 卡片内部统一使用左对齐 |

### Prompt 关键词

```
"左对齐文本"
"居中对齐的标题"
"网格对齐的布局（12 列网格）"
"整齐的排版，使用隐形网格线"
"统一的边缘对齐"
"表单标签和输入框左边缘对齐"
"使用 8px 基础网格"
"数字和价格右对齐"
```

## 4. 亲密性 (Proximity)

### 什么是它？

相关的元素靠近，不相关的元素分开。**空间距离暗示了信息的逻辑关系**——靠得近的元素被大脑自动视为一组，这是格式塔心理学的核心原则之一。

### 核心规则

> 元素之间的间距应该反映它们的关系：关系越近，间距越小。

### 应用示例

| 场景 | 做法 | 间距参考 |
|------|------|---------|
| **表单** | 标签紧靠输入框，不同字段组之间留大间距 | 标签到输入框 4px，字段间 16px，组间 32px |
| **按钮组** | 相关按钮靠近，不同组之间分开 | 组内 8px，组间 24px |
| **卡片** | 卡片内部元素紧凑，卡片之间留大间距 | 内部 12px，卡片间 24px |
| **导航** | 同级菜单项靠近，不同分类之间加分隔 | 项间 4px，分类间 16px+分隔线 |
| **内容区** | 段落内行距紧凑，段落之间间距大 | 行高 1.5，段间 24px |

### 间距层次

```
  4px   ■    →  紧密关联（标签与输入框）
  8px   ■■   →  相关元素之间（按钮组内）
 16px   ■■■  →  同一模块内部（表单字段间）
 24px   ■■■■ →  模块之间（信息卡片间）
 32px   ■■■■■ → 区块之间（表单组间）
 48px+  ■■■■■■ → 页面大区块之间（页头与内容区）
```

### 文字示意：间距如何传达分组

```
  好的间距                         差的间距

  ┌ 个人信息 ─────────┐            姓名
  │ 姓名               │            ┌────────┐
  │ ┌────────┐         │            └────────┘
  │ └────────┘         │  4px
  │              16px  │            邮箱
  │ 邮箱               │            ┌────────┐
  │ ┌────────┐         │            └────────┘
  │ └────────┘         │  16px          16px
  └────────────────────┘
         32px                       收货地址
  ┌ 地址信息 ─────────┐            ┌────────┐
  │ 收货地址           │            └────────┘
  │ ┌────────┐         │  16px
  │ └────────┘         │
  └────────────────────┘            城市
                                   ┌────────┐
  ↑ 用户一眼看出两组              └────────┘
    信息的边界
                                   ↑ 所有间距相同
                                     看不出分组关系
```

### 常见错误

| 错误 | 为什么是错的 | 怎么修 |
|------|-------------|--------|
| 所有元素间距都一样 | 看不出分组关系，信息像一团 | 使用 3-4 级间距梯度表达层级 |
| 标签离输入框太远 | 用户不确定标签对应哪个输入框 | 标签到输入框间距 ≤ 8px |
| 标签离上一个输入框太近 | 标签看起来属于上一个字段 | 确保标签距上方元素 > 距下方输入框 |
| 只用分隔线来分组 | 分隔线是补充手段，不应替代间距 | 先用间距分组，必要时再加分隔线 |

### Prompt 关键词

```
"相关的元素分组，组内间距小，组间间距大"
"清晰的模块划分"
"适当的间距层次（4/8/16/24/32/48px）"
"信息分组明确"
"标签紧靠对应的输入框"
"用间距而非分隔线来区分内容区块"
"格式塔亲密性原则"
"卡片内部紧凑，卡片之间留空"
```

## 5. 对比 (Contrast)

### 什么是它？

让元素之间产生**明显的差异**来突出重要内容、创建视觉趣味。对比的核心原则：**要么完全一样，要么完全不同**——微妙的差异看起来像是失误而非设计。

### 对比类型

| 类型 | 应用 | 示例 | 效果 |
|------|------|------|------|
| **颜色对比** | 深色背景 + 浅色文字 | 深色模式界面、CTA 按钮 | 最直接的视觉冲击 |
| **大小对比** | 大标题 + 小正文 | Hero 区域、数据大屏 | 建立信息层级 |
| **粗细对比** | 粗体标题 + 常规正文 | 文章页面、卡片 | 区分标题和内容 |
| **形状对比** | 圆形头像 + 方形卡片 | 社交产品、个人页面 | 增加视觉趣味 |
| **明暗对比** | 高亮区域 + 灰色背景 | 定价页"推荐"方案 | 引导注意力 |
| **密度对比** | 紧密内容区 + 大面积留白 | 落地页、产品介绍 | 呼吸感和节奏感 |

### 对比度标准（无障碍）

| 场景 | 最低对比度 | 标准 |
|------|-----------|------|
| 正文文字（<18px） | 4.5:1 | WCAG AA |
| 大号文字（≥18px 或 14px 粗体） | 3:1 | WCAG AA |
| 非文字元素（图标、边框） | 3:1 | WCAG AA |
| 增强标准 | 7:1 | WCAG AAA |

### 文字示意：按钮对比层级

```
  高对比（主操作）      中对比（次操作）      低对比（辅助操作）

  ┌──────────────┐    ┌──────────────┐
  │ ████████████ │    │  确认取消     │      取消
  │ ██ 立即购买 █│    │              │
  │ ████████████ │    └──────────────┘
  └──────────────┘
   填充色 + 白色文字     边框 + 深色文字       纯文字链接
   ← 最醒目               ← 可见但不抢眼       ← 最低调
```

### Before & After

**Before（对比不足）：**
```
┌──────────────────────────────────┐
│                                  │
│  专业版   ¥99/月                 │  浅灰色背景
│  ✓ 功能A  ✓ 功能B  ✓ 功能C       │
│  [  选择方案  ]                  │  灰色边框按钮
│                                  │
│  企业版   ¥299/月                │  浅灰色背景（一样）
│  ✓ 功能A  ✓ 功能B  ✓ 功能C+      │
│  [  选择方案  ]                  │  灰色边框按钮（一样）
│                                  │
└──────────────────────────────────┘
  ↑ 两个方案看起来没有区别，无法引导选择
```

**After（对比鲜明）：**
```
┌──────────────────────────────────────────────┐
│                       │ ████████████████████ │
│  专业版   ¥99/月      │ ██  企业版（推荐）██ │
│                       │ ██  ¥299/月       ██ │
│  ✓ 功能A              │ ██                ██ │
│  ✓ 功能B              │ ██ ✓ 功能A        ██ │
│  ✓ 功能C              │ ██ ✓ 功能B        ██ │
│                       │ ██ ✓ 功能C+       ██ │
│  [  选择方案  ]       │ ██                ██ │
│                       │ ██ [■ 立即选择]   ██ │
│                       │ ████████████████████ │
└──────────────────────────────────────────────┘
  ↑ 推荐方案用深色背景+填充按钮，形成强对比
```

### 常见错误

| 错误 | 为什么是错的 | 怎么修 |
|------|-------------|--------|
| 按钮和背景颜色太接近 | 用户看不清、找不到操作入口 | 主按钮与背景对比度至少 3:1 |
| 微妙的差异（如 #333 vs #444） | 看起来像失误而非有意设计 | 要么一样，要么差异明显 |
| 所有按钮都是高对比 | 失去层级区分，用户不知道点哪个 | 只给主操作高对比，次操作降低 |
| 忽视色盲用户 | 约 8% 男性有色觉障碍 | 不仅靠颜色区分，同时用形状/图标/文字 |
| 浅灰色文字在白色背景上 | 不满足无障碍标准，难以阅读 | 检查对比度是否达到 4.5:1 |

### Prompt 关键词

```
"高对比度的配色，满足 WCAG AA 标准"
"大小对比突出标题"
"清晰的视觉区分"
"重要元素使用强调色，次要元素使用低饱和色"
"主次分明的按钮层级：填充、边框、文字链接"
"推荐方案使用高对比背景突出"
"要么完全一样，要么完全不同"
"确保色盲用户也能区分关键信息"
```

---

## 五大原则速记表

| 原则 | 核心 | 一句话口诀 | 检查问题 |
|------|------|-----------|---------|
| **视觉层次** | 引导视线 | 让用户 3 秒找到重点 | 用户知道先看哪里吗？ |
| **一致性** | 统一规范 | 相同功能 = 相同外观 | 相同功能看起来一样吗？ |
| **对齐** | 整齐有序 | 每个元素都要有"理由"在那 | 元素边缘对齐了吗？ |
| **亲密性** | 分组明确 | 近的是一组，远的是另一组 | 相关元素在一起吗？ |
| **对比** | 突出重点 | 要么一样，要么完全不同 | 重要内容够突出吗？ |

## 原则之间的关系

这 5 个原则不是孤立的，它们互相配合：

```
  视觉层次 ← 使用 → 对比（通过对比创造层次）
     ↑                  ↑
     │                  │
  对齐（对齐强化层次）   亲密性（分组配合对比）
     │                  │
     └──── 一致性 ──────┘
          （一致性是所有原则的基础）
```

- **视觉层次 + 对比**：用大小和颜色对比来创造信息层级
- **亲密性 + 对齐**：用间距分组，用对齐线串联
- **一致性**：贯穿所有原则，确保规则被统一执行

## 自检清单

设计完成后，用这个清单快速检查：

- [ ] **视觉层次**：页面有明确的视觉焦点，用户 3 秒内能找到核心信息
- [ ] **视觉层次**：字号至少有 3 级梯度，每页只有 1 个主 CTA
- [ ] **一致性**：相同类型的按钮在所有页面外观一致
- [ ] **一致性**：间距使用 8px 倍数系统
- [ ] **对齐**：所有元素都在网格线上，没有"随意摆放"的元素
- [ ] **对齐**：表单标签和输入框左边缘对齐
- [ ] **亲密性**：相关元素间距 < 不相关元素间距
- [ ] **亲密性**：标签离自己的输入框比离上一个输入框更近
- [ ] **对比**：主按钮与背景对比度 ≥ 3:1
- [ ] **对比**：正文与背景对比度 ≥ 4.5:1（WCAG AA）
