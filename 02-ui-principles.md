# 02 - UI 设计核心原则

> 掌握这 5 个原则，让你的界面设计立即提升一个档次。

---

## 1. 视觉层次 (Visual Hierarchy)

### 什么是它？
引导用户先看什么、后看什么的设计方法。

### 实现方法

| 方法 | 说明 | 示例 |
|------|------|------|
| **大小** | 重要的元素更大 | 标题 > 副标题 > 正文 |
| **颜色** | 重要的元素更醒目 | 主按钮用品牌色 |
| **对比** | 高对比度吸引注意力 | 深色文字配浅色背景 |
| **留白** | 给重要元素周围留更多空间 | 标题周围留白更多 |
| **位置** | 屏幕上方和左侧更醒目 | 核心功能放左上角 |

### Prompt 关键词

```
"清晰的视觉层次"
"重要信息突出显示"
"引导用户视线从上到下"
"主按钮使用高对比色"
"标题使用更大字号"
```

### 实际应用

```
❌ 错误：所有文字一样大，用户不知道先看哪里
✅ 正确：标题 32px，副标题 24px，正文 16px
```

## 2. 一致性 (Consistency)

### 什么是它？
相同的功能看起来、用起来都一样。

### 为什么重要？
- 减少用户学习成本
- 建立信任感
- 提高开发效率

### 需要一致的元素

| 元素 | 一致的内容 |
|------|-----------|
| **按钮** | 主按钮、次按钮、危险按钮的样式 |
| **字体** | 大小、颜色、字重 |
| **间距** | 8px 倍数系统（8, 16, 24, 32...） |
| **圆角** | 统一使用 4px 或 8px |
| **图标** | 线型或面型，统一风格 |

### Prompt 关键词

```
"保持一致的设计风格"
"使用统一的按钮样式"
"遵循设计系统规范"
"相同的交互模式"
"统一的间距系统"
```

### 实际应用

```
❌ 错误：页面 A 的保存按钮是蓝色，页面 B 是绿色
✅ 正确：所有保存按钮都是蓝色，删除按钮都是红色
```

---

## 3. 对齐 (Alignment)

### 什么是它？
元素之间的位置关系。

### 对齐方式选择

| 方式 | 适用场景 | 示例 |
|------|---------|------|
| **左对齐** | 大段文字阅读 | 文章正文、表单标签 |
| **居中对齐** | 标题、短文本 | 页面标题、按钮文字 |
| **右对齐** | 数字、价格 | 表格数字列 |
| **两端对齐** | 正式文档 | 印刷品、合同 |

### Prompt 关键词

```
"左对齐文本"
"居中对齐的标题"
"网格对齐的布局"
"整齐的排版"
"统一的边缘对齐"
```

### 实际应用

```
❌ 错误：表单标签有的左对齐，有的右对齐
✅ 正确：所有表单标签统一左对齐，输入框左边缘对齐
```

---

## 4. 亲密性 (Proximity)

### 什么是它？
相关的元素靠近，不相关的元素分开。

### 应用示例

| 场景 | 做法 |
|------|------|
| **表单** | 标签靠近输入框，不同表单组之间留大间距 |
| **按钮组** | 相关按钮靠近（间距 8px），不同组之间留大间距（间距 24px） |
| **卡片** | 卡片内部元素紧凑，卡片之间留大间距 |

### 间距层次

```
小间距 (8px)  →  相关元素之间
中间距 (16px) →  模块内部
大间距 (24-32px) →  模块之间
超大间距 (48px+) →  页面区块之间
```

### Prompt 关键词

```
"相关的元素分组"
"清晰的模块划分"
"适当的间距层次"
"信息分组明确"
"相关元素靠近"
```

### 实际应用

```
❌ 错误：所有元素间距都一样，看不出分组
✅ 正确：姓名字段间距 8px，个人信息组与地址组间距 32px
```

## 5. 对比 (Contrast)

### 什么是它？
让元素之间产生差异，突出重要内容。

### 对比类型

| 类型 | 应用 | 示例 |
|------|------|------|
| **颜色对比** | 深色背景 + 浅色文字 | 深色模式界面 |
| **大小对比** | 大标题 + 小正文 | 文章页面 |
| **粗细对比** | 粗体标题 + 常规正文 | 层级区分 |
| **形状对比** | 圆角卡片 + 直角按钮 | 视觉区分 |

### 对比度标准

- **文字与背景**：至少 4.5:1（WCAG AA 标准）
- **重要元素**：使用高对比色吸引注意
- **次要元素**：降低对比度，不抢注意力

### Prompt 关键词

```
"高对比度的配色"
"大小对比突出标题"
"清晰的视觉区分"
"重要元素使用强调色"
"主次分明的层级"
```

### 实际应用

```
❌ 错误：按钮和背景颜色太接近，看不清
✅ 正确：主按钮使用品牌色，与背景形成明显对比
```

## 五大原则速记表

| 原则 | 核心 | 检查问题 |
|------|------|---------|
| **视觉层次** | 引导视线 | 用户知道先看哪里吗？ |
| **一致性** | 统一规范 | 相同功能看起来一样吗？ |
| **对齐** | 整齐有序 | 元素边缘对齐了吗？ |
| **亲密性** | 分组明确 | 相关元素在一起吗？ |
| **对比** | 突出重点 | 重要内容够突出吗？ |
