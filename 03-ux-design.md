# 03 - UX 用户体验设计

> 让用户用得爽，而不只是看得爽。

---

## 1. 用户旅程 (User Journey)

### 什么是它？
用户完成一个任务的完整流程。

### 设计步骤

```
1. 用户目标是什么？
   ↓
2. 需要哪些步骤？
   ↓
3. 每一步需要什么信息？
   ↓
4. 可能的错误和解决方案？
   ↓
5. 成功后的反馈？
```

### 示例：电商购买流程

| 步骤 | 用户目标 | 页面 | 关键元素 |
|------|---------|------|---------|
| 1 | 找到商品 | 商品列表 | 搜索、筛选、商品卡片 |
| 2 | 了解详情 | 商品详情 | 图片、价格、描述、评价 |
| 3 | 确认购买 | 购物车 | 商品清单、数量、总价 |
| 4 | 完成支付 | 结算页 | 地址、支付方式、确认 |
| 5 | 获得确认 | 成功页 | 订单号、预计送达、追踪 |

### Prompt 关键词

```
"清晰的用户流程"
"减少操作步骤"
"明确的下一步指引"
"流畅的交互体验"
"完整的用户旅程"
```

## 2. 信息架构 (Information Architecture)

### 什么是它？
组织和结构化信息的方式。

### 核心问题

1. **用户来找什么？** → 确定核心内容
2. **如何分类最合理？** → 建立分类体系
3. **如何导航最方便？** → 设计导航结构

### 常用结构

| 结构 | 适用场景 | 示例 |
|------|---------|------|
| **层级结构** | 内容有明确父子关系 | 文件管理器、组织架构 |
| **矩阵结构** | 多维度筛选 | 电商商品筛选 |
| **线性结构** | 步骤流程 | 注册流程、结账流程 |
| **自然结构** | 探索式内容 | 相册、社交动态 |

### 导航设计原则

- **7±2 原则**：导航项不超过 9 个
- **层级不超过 3 层**：用户容易迷失
- **面包屑导航**：显示当前位置

### Prompt 关键词

```
"清晰的信息架构"
"直观的导航结构"
"合理的分类组织"
"易于查找的布局"
"层级清晰的导航"
```

## 3. 可用性原则 (Usability)

### 尼尔森十大可用性原则（简化版）

| # | 原则 | 说明 | 示例 |
|---|------|------|------|
| 1 | **系统状态可见** | 告诉用户发生了什么 | 加载动画、进度条 |
| 2 | **贴近真实世界** | 用用户熟悉的语言 | "保存" 而不是 "提交表单" |
| 3 | **用户控制** | 允许撤销操作 | 删除前有确认，可恢复 |
| 4 | **一致性** | 遵循平台惯例 | iOS/Android 原生规范 |
| 5 | **防错设计** | 预防错误发生 | 日期选择器代替手动输入 |
| 6 | **识别而非回忆** | 让用户看到选项 | 下拉菜单而不是输入 |
| 7 | **灵活高效** | 兼顾新手和专家 | 快捷键、批量操作 |
| 8 | **美观简洁** | 去除无关信息 | 每页一个主要目标 |
| 9 | **错误提示** | 清晰的错误信息 | 告诉用户错在哪、怎么改 |
| 10 | **帮助文档** | 易于搜索和查找 | 上下文帮助、FAQ |

### 错误提示设计

**好的错误提示包含：**
1. 发生了什么？
2. 为什么会发生？
3. 怎么解决？

```
❌ 错误："操作失败"
✅ 正确："密码太短（发生了什么）。密码需要至少 8 位（为什么）。请重新输入更长的密码（怎么解决）。"
```

### Prompt 关键词

```
"清晰的反馈提示"
"错误预防设计"
"直观的操作方式"
"简洁的界面"
"用户可控的操作"
```

## 4. 无障碍设计 (Accessibility)

### 什么是它？
让所有人都能使用你的产品，包括残障人士。

### 关键要点

| 要点 | 标准 | 检查方法 |
|------|------|---------|
| **颜色对比度** | 至少 4.5:1 | 使用对比度检查工具 |
| **不只靠颜色** | 用图标+文字传达信息 | 红绿色盲测试 |
| **键盘操作** | 所有功能可用键盘完成 | Tab 键测试 |
| **图片 Alt 文本** | 描述图片内容 | 屏幕阅读器测试 |
| **字体大小** | 至少 16px | 可读性测试 |

### 常见无障碍问题

```
❌ 错误：只用红色表示错误
✅ 正确：红色 + 错误图标 + 文字说明

❌ 错误：按钮太小，难以点击
✅ 正确：点击区域至少 44x44px

❌ 错误：图片没有 alt 文本
✅ 正确：所有图片都有描述性 alt 文本
```

### Prompt 关键词

```
"无障碍设计"
"高对比度配色"
"清晰的文字可读性"
"支持键盘导航"
"屏幕阅读器友好"
```

## UX 设计检查清单

- [ ] 用户能轻松找到核心功能吗？
- [ ] 每个操作都有反馈吗？
- [ ] 错误提示清晰有用吗？
- [ ] 用户能撤销操作吗？
- [ ] 界面是否简洁无干扰？
- [ ] 颜色对比度足够吗？
- [ ] 支持键盘操作吗？
- [ ] 加载状态有提示吗？
